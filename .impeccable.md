## Design Context

### Users
This local app is for a solo human player playing Diplomacy against LLM-controlled powers. The interface should be understandable to non-technical players who know board games but do not know the underlying engine, order syntax, or LLM plumbing.

### Brand Personality
Strategic, readable, and board-game-like. It should feel like a polished tabletop strategy game running locally, not a developer dashboard or experiment runner.

### Aesthetic Direction
Full-screen illustrated map as the primary surface, with lightweight floating controls for chat, orders, score, and turn history. Favor clear country color, flags, unit symbols, and map gestures over raw order strings. Technical state such as model provider or run directory should be visible only when it helps the player recover from a problem.

### Design Principles
- Board first: the map is the game; panels should support it rather than compete with it.
- Plain-language play: show "Fleet to Belgium" before engine syntax like `F ENG - BEL`.
- Real AI only: chat and orders must come from live LLM calls unless explicitly running a development mock.
- Private diplomacy: each agent sees only legitimate public information, its own private messages, its diary, and board state.
- Progressive detail: expose the simplest next action first, while preserving advanced Diplomacy orders through fallback controls.
